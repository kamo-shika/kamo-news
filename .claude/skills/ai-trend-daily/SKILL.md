# ai-trend-daily

## このスキルの目的

毎朝 7:00 JST に当日の AI トレンドニュース記事 `news/YYYY-MM-DD.md` を生成する。
読者は朝起きてリポジトリを開いた時に、その日読むべき AI 情報を 5〜10 件のトピックで把握できる。

## 必ず最初に読む参照

- `references/sources.md` — 情報源の一覧と取得方法
- `references/output-format.md` — 出力 Markdown のフォーマット
- `references/validation.md` — セルフレビュー チェックリストと再生成判定

## 実行手順

### Step 1: 日付の決定

実行日 (JST) を `YYYY-MM-DD` 形式で取得する。出力ファイル名はこの日付に基づく。

```bash
date -u -d '+9 hours' +%Y-%m-%d
```

### Step 2: 情報収集

`references/sources.md` に記載された各情報源から、過去 24 時間 (UTC 基準) のエントリを取得する。
取得は `curl` で行い、JSON は `jq` で、RSS/Atom は適切にパースする。

各情報源で取得失敗してもジョブ全体は止めず、取得できた範囲で進める。

### Step 3: フィルタリングと選出

取得したエントリから、以下の条件で AI 関連トピックを抽出する:

- LLM / 生成 AI / エージェント / AI コーディング / マルチモーダル / RAG / ファインチューニング 等の話題
- 一般的な機械学習・データサイエンス記事は対象外 (生成 AI 文脈に限定)
- 同一トピックは 1 件に統合 (異なる情報源で同じ話題は最も詳しい出典を採用)

重要度を判定し、上位 5〜10 件を選出する。重要度の判断軸:
- 公式リリース (新モデル発表、メジャーアップデート) > 議論 > 個人ブログ
- 一次情報を含む > 二次的なまとめ記事

### Step 4: 記事生成

`references/output-format.md` のテンプレートに従い、`news/YYYY-MM-DD.md` を生成する。

各トピックには必ず:
- 3〜5 文の要約 (日本語)
- 出典 URL (一次情報を優先)
- カテゴリ (論文 / 公式リリース / 議論 / ツール 等)

を含める。

### Step 5: セルフレビュー

`references/validation.md` のチェックリストに従い、生成した記事を自己評価する。

全 8 項目を順にチェックし、判定結果を記録する。

### Step 6: 再生成判定

- すべて Pass → Step 7 (完了報告) へ
- いずれか Fail かつ再生成回数 < 1 → Fail 項目を明示してもう一度 Step 2〜5 を実行 (再生成 1 回)
- 2 回目でも Fail → 完了報告に Fail 情報を含めて終了 (PR は呼び出し側が draft で作成)

### Step 7: 完了報告

以下を報告する:

- 生成したファイルパス
- 選出件数
- セルフレビュー結果 (Pass / Fail、Fail の場合は項目)
- 再生成の有無
- `needs-review` ラベル付与の要否 (= 最終結果が Fail なら true)

## 制約

- 出典 URL は実在するものに限る。架空の URL は絶対に書かない。
- 取得したエントリのドメインと一致しない URL を出さない。
- 「情報を取得できませんでした」「APIエラー」等のシステムメッセージを記事本文に含めない。
- 各トピックの本文は最低 100 文字。
