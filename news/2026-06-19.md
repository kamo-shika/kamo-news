### 2. Anthropic の「Claude Mythos」と SK Telecom を巡る輸出規制問題

### 3. MCP の Enterprise-Managed Authorization 拡張が安定版に

Model Context Protocol (MCP) の Enterprise-Managed Authorization (EMA) 拡張が安定版に達した。企業は Okta などのアイデンティティプロバイダーを通じて MCP サーバーへのアクセスを一元管理でき、ユーザーは初回ログイン時に接続済みの MCP サーバーへ自動的にアクセスできるようになる。アプリごとに OAuth の同意を繰り返す煩雑さが解消され、エンタープライズ環境への MCP 本格展開を大きく後押しする。Anthropic・Microsoft・Okta がすでに採用を表明しており、対応 MCP サーバーも急速に増えている。

### 4. Agentic Resource Discovery Specification (ARD) 公開

AI エージェントが利用可能なツール・スキル・MCP サーバー・API などのリソースを自動発見するための標準仕様「Agentic Resource Discovery (ARD)」が公開された。AI クライアントが「このタスクに対応できるリソースはどれか」と一問一答形式で問い合わせると、一致するリソースの一覧・説明・接続先が返ってくる設計になっている。ARD はあくまで「発見」のみを担当し、実際のツール呼び出しは MCP や API など各リソース固有のメカニズムに委ねられる。GitHub の Agent Finder と Hugging Face の Discover がすでに ARD を実装した先行サービスとして稼働しており、エンタープライズ向けエージェント展開の加速が期待される。

- 出典: <https://agenticresourcediscovery.org/introduction/>
- 出典 (Google アナウンス): <https://developers.googleblog.com/en/announcing-the-agentic-resource-discovery-specification/>

### 6. 「RTK のトークン削減は幻想」— コーディングエージェント向けツールへの懐疑論

「60〜90% のトークン削減」を謳う LLM コーディングエージェント向けの Terminal 出力圧縮ツール「RTK」(GitHub スター 6 万超) に対し、その主張が根本的に誤解を招くものだとする批評記事が Hacker News で話題を呼んだ。RTK はコマンドライン出力のみを圧縮対象とするが、実際の API コストの主因はファイル読み込み・リポジトリコンテキスト・システムプロンプト・モデル内部の推論トークンであり、RTK はこれらを一切削減しないと指摘されている。また GitHub Issues にはすでに、ターミナル出力が静かに破損・欠落する「サイレント失敗」の報告が上がっており、エージェントが誤情報に基づいて動作するリスクがある。可視化向けのバニティ指標に惑わされず、実際のコスト最適化を見極める視点を問い直す論考として注目される。

### 7. Rubric-Conditioned Self-Distillation: LLM 後学習の新フレームワーク

Yale 大学らが提案する「Rubric-Conditioned Self-Distillation」は、LLM の後学習 (Post-training) における報酬監督の課題を解決する新しいフレームワークだ。従来の Chain-of-Thought 蒸留はアノテーションコストが高くノイズを含みやすく、RL ではスカラー報酬信号がどの側面を改善すべきかを隠蔽するという問題があった。本手法は「ルーブリック (Rubric)」と呼ばれる構造化された多基準フィードバックを教師モデルに与え、学生モデル自身のサンプリング軌跡に対してトークンレベルのガイダンスを提供するオンポリシー自己蒸留を実現する。推論能力の向上を目指す LLM 後学習研究に新たな方向性を示す成果として、関連研究者の注目を集めている。
