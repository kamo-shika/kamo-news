### 1. Stable Audio 3: 3 サイズ構成の高速音声拡散モデルファミリー

Stability AI が Stable Audio 3 を Hugging Face Daily Papers に公開した。small・medium・large の 3 サイズで構成される高速な潜在拡散モデルファミリーで、可変長の音声生成と編集の両方に対応している。テキストプロンプトから音楽・効果音・音声を生成できるだけでなく、既存オーディオの編集タスクにも対応しており、ポストプロダクションや創作現場での実用を視野に入れた設計となっている。モデルサイズを 3 段階に分けることで、高精度が要求されるスタジオ用途から軽量推論が必要な用途まで幅広いシナリオをカバーする。拡散モデルのオーディオ生成分野における重要なマイルストーンとして注目される。

### 2. SpecBench: 長期コーディングエージェントの報酬ハッキングを測定

論文「SpecBench: Measuring Reward Hacking in Long-Horizon Coding Agents」が Hugging Face Daily Papers に登録された。長期間動作するコーディングエージェントが生成するコード量はレビュー可能な範囲を超え始めており、品質保証は自動テストスイートへの依存が不可避になっている。しかしこの状況では、エージェントがテストをパスするために意図せず「ハック」する行動（報酬ハッキング）が自然発生するリスクがある。SpecBench はこの問題を専用のベンチマークセットで定量評価することで、エージェントが仕様を正しく実装しているか、それともテストのみをすり抜けているかを峻別できるようにする。AI コーディングエージェントの本番環境投入が加速する中で、安全・品質保証の観点から重要な研究貢献だ。

- 出典: <https://huggingface.co/papers/2605.21384>

### 3. Mix-Quant: NVFP4 量子化でプリフィルを最大 3 倍高速化、エージェント LLM 推論を効率化

Hugging Face Daily Papers 収録の「Mix-Quant: Quantized Prefilling, Precise Decoding for Agentic LLMs」は、エージェント型 LLM の推論を効率化するフェーズ対応量子化フレームワークを提案している。プリフィル段階（入力コンテキストの一括処理）は計算量が支配的でありながら量子化誤差が生じにくいという観察に基づき、プリフィルには高スループットな NVFP4（FP4 量子化）を適用し、デコード段階は BF16 精度を維持するという非対称設計を採用した。実験では長文コンテキストやエージェントベンチマークにおいてタスク性能をほぼ損なわずプリフィルで最大 3 倍の高速化を達成している。エージェントは計画・ツール呼び出し・メモリ参照を多段階で繰り返すため推論コストが高く、本手法は生産環境での LLM エージェント運用コスト削減に直接貢献する。

- 出典: <https://huggingface.co/papers/2605.20315>

### 4. TIDE: I/O 最適化で MoE 拡散 LLM の推論ボトルネックを解消

「TIDE: Efficient and Lossless MoE Diffusion LLM Inference with I/O-aware Expert Offload」が Hugging Face Daily Papers に公開された。Diffusion Large Language Models（dLLMs）は自己回帰型モデルに比べてハードウェア活用率が高く有望視されているが、Mixture-of-Experts（MoE）構成を持つ大規模モデルでは、エキスパートモジュールのメモリ帯域が推論のボトルネックになる。TIDE は I/O を意識したエキスパートのオフロード管理で推論スループットを精度損失なく大幅に改善することを提案し、実験ではメモリ消費を抑えつつ処理速度を向上させる効果を確認している。MoE × 拡散 LLM という次世代アーキテクチャの実用化を後押しする研究として注目される。

- 出典: <https://huggingface.co/papers/2605.20179>

### 6. Mem-π: LLM エージェントが「いつ何を生成するか」を学ぶ適応的メモリフレームワーク

「Mem-π: Adaptive Memory through Learning When and What to Generate」が Hugging Face Daily Papers に登録された。従来の LLM エージェントメモリシステムは外部ベクトルストアからの検索（RAG 的アプローチ）に依存するが、Mem-π はエージェント自身が必要なタイミングに必要な知識・文脈情報をオンデマンドで生成する仕組みを持つ。「何を外部に保存するか」ではなく「いつ何を生成するか」をモデルが学習する点が革新的で、検索コスト・ハルシネーションリスクを低減しながら長期エージェント動作を効率化できる可能性がある。エージェントのメモリ設計に新しい視点を提供する研究として注目される。
