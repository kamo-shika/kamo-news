### 1. DiffusionGemma: 拡散モデルで GPU テキスト生成を最大 4 倍高速化

Google DeepMind が発表した「DiffusionGemma」は 26B パラメータの Mixture of Experts (MoE) モデルで、従来の自動回帰型言語モデルとは異なり 256 トークンのブロックを同時生成する拡散ベースのアーキテクチャを採用している。単一 H100 GPU で 1,000 トークン/秒以上を達成し、従来モデル比で最大 4 倍の高速化を実現した。双方向注意機構により、インライン編集やコード補完のような非線形タスクにも優れた柔軟性を持つ点も特徴だ。Apache 2.0 ライセンスで Hugging Face から公開されており、開発者はローカル GPU 環境での低遅延インタラクティブ AI アプリケーション構築に活用できる。

- 出典: <https://deepmind.google/blog/diffusiongemma-4x-faster-text-generation/>
- カテゴリ: 公式リリース

### 2. MosaicLeaks: 研究エージェントの「モザイク効果」による情報漏洩リスクを定量化

ServiceNow が Hugging Face ブログで発表した MosaicLeaks は、AI 研究エージェントが機密文書と外部ウェブ検索を組み合わせる際に生じる情報漏洩リスクを定量化したベンチマークだ。個々のウェブ検索クエリは無害に見えても、それらを組み合わせると「モザイク効果」により機密情報が第三者に推測可能になる危険性を実証した。研究では「プロンプト指示だけではプライバシーを保証できない」ことが示されており、タスク成功を優先すると漏洩率が上昇する傾向も確認された。提案する PA-DR (Privacy-Aware Deep Research) 法により、タスク成功率をほぼ維持しながら漏洩率を 34.0% から 9.9% まで削減することに成功している。

- 出典: <https://huggingface.co/blog/ServiceNow/mosaicleaks>
- カテゴリ: 論文

### 3. DeepMind が AI エージェントセキュリティの「AI Control Roadmap」を公表

Google DeepMind が「Securing the future of AI agents」を公開し、能力向上が続く AI エージェントを内部脅威として扱う多層防御アーキテクチャを提案した。MITRE ATT&CK フレームワークを参考に AI エージェントの攻撃パターンを分類し、サンドボックス化・エンドポイント保護・プロンプトインジェクション対策に加え、信頼できる AI システムが他のエージェントを監視するという独自のコントロール層を設ける構成を示している。100 万件以上のコーディングタスク分析から、フラグされたイベントの大半はエージェントの意図的な悪用ではなく「誤解釈」に起因することも判明した。モデル能力の向上に合わせてセキュリティ対策を段階的に強化するロードマップも示されており、AI をプロダクションに導入する組織への実践的指針として注目される。

### 4. FAPO: Claude Code がオーケストレーターとなるマルチステップ LLM パイプライン全自動最適化

「FAPO (Fully Autonomous Prompt Optimization)」は、Claude Code をオーケストレーターとして活用し、LangGraph ベースの LLM パイプライン全体を 6 段階のループで自動最適化するフレームワークだ。評価実行 → 故障属性化 → 変更提案 → 独立レビュー → 性能比較 → 反復/エスカレーション という流れで、まずプロンプト編集から着手し、改善が停滞した場合のみチェーン構造の変更にエスカレーションする「プロンプトファースト」戦略を採用する。スコープ管理が 3 段階 (プロンプトのみ/チェーンパラメータ/チェーン構造) に分かれており、タスクごとの許可範囲を定義できる。6 つのベンチマークで先行手法 GEPA の 18 比較中 15 で上回り、HoVer と IFBench では構造変更が必要と判断され +24〜49 ポイントの大幅改善を確認している。

- 出典: <https://huggingface.co/papers/2606.19605>

### 5. LoRA 一強時代に終止符? OFT が画像生成タスクで上回る PEFT 比較結果

Hugging Face が公開した PEFT (Parameter-Efficient Fine-Tuning) 技術の比較ブログで、LoRA が必ずしも最良の選択ではないことが示された。Hugging Face Hub のモデルの 98.4% が LoRA を採用しているが、これは性能優位性よりも普及の自己強化的なサイクルによる部分が大きいと指摘する。LLM の数学タスクでは LoRA も良好な性能を示すが「パレート最適曲線」上の他の手法と同等であり、画像生成タスクでは OFT (Orthogonal Fine-Tuning) が「より高い精度かつ低いメモリ使用量」で LoRA を凌駕した。PEFT ライブラリが統一 API で複数手法を提供しているため技術切り替えのコストは低く、タスクに応じた手法選択の重要性を再確認する内容として話題を呼んでいる。

- 出典: <https://huggingface.co/blog/peft-beyond-lora>
