### 1. Cloudflare が AI エージェント向け「一時アカウント」機能をリリース — 60 分間の無認証デプロイを実現

Cloudflare が `wrangler deploy --temporary` フラグによる「一時アカウント」機能を公開し、AI エージェントが OAuth フローや MFA なしに Workers をデプロイできるようにした。従来、バックグラウンドで動作する自動化エージェントはサインアップ画面やダッシュボード操作の壁に直面していたが、この機能により最大 60 分間の一時プレビュー環境が即座に払い出される。デプロイ後はエージェントがコードの検証・再デプロイを繰り返し、完了後に人間が claim URL を開くだけで正規アカウントに昇格できる設計だ。Workers・KV・D1・Durable Objects・Queues など主要プロダクトが対応しており、AI エージェントがインフラを自律的にプロビジョニングする「Agentic Cloud」構築への重要な一歩として Hacker News で 130 以上のポイントを集めた。

### 2. LedgerAgent — ポリシー違反を事前ブロックする構造化状態管理付きツール呼び出しエージェント

カスタマーサービスなどビジネスドメインで動作するツール呼び出しエージェント向けの「LedgerAgent」が Hugging Face Daily Papers で発表された。従来のエージェントはポリシーに関係する情報をすべてプロンプトに積み込み、モデルが毎回状態を再構築していたが、LedgerAgent は観察済みのタスク状態を専用の「台帳」として分離管理する。台帳はツール呼び出し実行前に状態依存のポリシー制約を照合し、違反が見込まれる場合はコール自体をブロックする設計で、「正しいファクトを取得しても古い・不完全な情報で意思決定する」「構文的に正しいツール呼び出しがポリシーに違反する」という 2 つの典型的な障害モードに対処する。4 つのカスタマーサービスドメインでオープン・クローズド双方のモデルを対象にテストし、従来のプロンプトベースアプローチより平均 pass@k が向上することを確認している。

### 3. Multi-LCB — Python 偏重だった LLM コード評価を 12 言語に拡張した新ベンチマーク

LiveCodeBench (LCB) は LLM のコード生成能力評価で事実上の標準となっているが、Python 専用という制約があった。Multi-LCB はこの Python 課題を Go・Java・Rust・Kotlin・Swift・TypeScript など 12 言語の等価な課題に変換し、LCB と同一の汚染対策および評価プロトコルを維持した拡張ベンチマークだ。24 モデルを対象とした評価では、Python 過学習の証拠、言語固有の汚染、多言語間での大幅な性能格差が明らかになり、特定の言語に最適化されたモデルが他言語では著しく性能低下することが実証された。将来の LCB 更新と自動的に同期する設計になっており、LLM の多言語コード生成能力を継続的に追跡できる点も特徴だ。

### 5. UFP4 — E2M1 の「縮小バイアス」を排した均一グリッドによる FP4 LLM プレトレーニング

NVIDIA Blackwell など最新 GPU が採用する FP4 (E2M1 形式) で LLM を学習する際の根本的な問題「縮小バイアス (Shrinkage Bias)」を特定し、均一グリッド形式 (E1M2/INT4) で解決する「UFP4」レシピを提案した論文が Hugging Face Daily Papers に掲載された。E2M1 の非均一な表現ビンは幾何学的非対称性に起因する系統的な負の丸め誤差を生じさせ、アウトライアを分散させる Random Hadamard Transform (RHT) と組み合わせると学習不安定性がさらに増幅されることを理論・実験の両面から示している。Dense 1.5B・MoE 7.9B・MoE 124B と複数規模のモデルでの長期プレトレーニング実験で、UFP4 は既存 E2M1 ベースラインより BF16 比の損失劣化が一貫して小さいことを確認した。将来の GPU アーキテクチャが E1M2/INT4 スタイルの均一 4 ビットグリッドを第一級サポートとして追加すべきという提言も含んでいる。

### 7. FlowBender — 推論時フィードバックで整合性を高める自己修正型条件付き生成フレームワーク

条件付き拡散・フローモデルが自らが定義する制約を満たせないという根本問題に取り組む「FlowBender」が発表された。深度マップ条件付きモデルが出力画像から再抽出した深度を入力と一致させられない例を起点に、「モデルは自身の整合性エラーを用いて学習したことがない」という問題意識から出発している。閉ループフレームワークとして設計されており、各ステップで「先読みパス」によるクリーン信号推定 → 順方向演算子による偏差計算 → 修正速度を算出する「修正パス」という流れで動作し、微分不可能な演算子向けのゼロ次変形と追加コストを最小化する「前ステップショートカット」も提案している。画像間変換・復元・3D メッシュテクスチャリングのすべてで監督あり・ガイダンスベースのベースラインを上回る結果を示した。
